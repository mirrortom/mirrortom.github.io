概述

插件是wenpublish插件的vscode版本,使用发服务和vs版本是同一个服务.已有的vs插件功能都有,另外新增加了文件合并和stylus编译功能.

使用vscode做前端项目,比如js/node/ts/html/css还有一些脚本语言是比较合适的,主要是感觉vs启动比较慢,还有这些类型的语言不需要编译,而vs还是用于服务端比较好.

用nodejs/ts开发vscode的插件,和用c#开发vs插件,感觉vscode的插件根容易一些,项目复杂度远远低于vs插件.

有个功能没实现,获取项目资源管理器窗口中,选定的文件.这个没有API啊!!

使用

alt+1,1 发布选中文件(只能使用鼠标右键,因为没有找到获取选中文件的API)

alt+2,2 发布选中目录(只能使用鼠标右键,因为没有找到获取选中文件的API)

alt+3,3 发布整个工作区,只支持第一个工作区

alt+4,4 合并文件,整个工作区范围内,根据配置文件

alt+5,5 编译stylus,整个工作区范围内,根据配置文件

ctrl+s 按保存时,会执行文件合并和stylus编译,根据文件的扩展名,还有配置文件设置.

功能

编译razor文件和vs版本插件一样

合并文件,将多个文件合并到一个文件,一般用于js

编译stylus成css

配置文件

配置文件在vs版本上增加了bundle和stylus节点,用于文件合并和编译stylus.

...
"bundle": [
    {
        "outputFile": "src/asset/js/index.js",
        "inputFiles": [
            "src/asset/js/1.part.js",
            "src/asset/js/2.part.js",
            "src/asset/js/3.part.js",
            "src/asset/js/4.part.js",
            "src/asset/js/5.part.js"
        ]
    }
],
"stylus":[
    {
        outputFile:"index.css",
        inputFile:"index.styl"
    }
]

常用的vscode插件

vs下的Bundle&Mini插件,还有web compile插件可以提供网页端js/css/html的所有功能.在vscode下的插件更加丰富了.

如果没有找到,由于nodejs丰富的库,还可以自己写.比如由于没有找到BundleMini和web Compile的vscode版本,发现可以使用stylus的jsapi实现编译.
